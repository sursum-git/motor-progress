Resumo operacional curto

1. O que e este projeto

Este repositorio implementa um motor de consulta dinamica em Progress/OpenEdge com exposicao via PASOE/REST e uma UI web Kendo para operar o motor.

Backend OOABL para validacao, plano, execucao, fila async e escrita de resultado.
Frontend Aplicacao web para contexto, consultas, metadados e configuracao operacional.

2. Objetivo funcional

Permitir consultas estruturadas sem SQL/ABL livre, com validacao, joins, paginacao, execucao sincrona e assincrona e contexto operacional por cliente, ambiente e empresa.

  • fontes e aliases
  • joins controlados
  • filtros validados
  • fila persistente
  • resultado JSON
  • contexto funcional persistido

10. Contrato atual da API

Base: http://localhost:8890/web/SursumDynamicQuery

Endpoints principais

  • POST /query
  • POST /jobs/drain
  • GET /jobs/{jobId}
  • GET /jobs/{jobId}/result

Formato assumido

sources[].nome
sources[].alias
sources[].campos
select[].sourceAlias
select[].field
select[].outputAlias
filters[].sourceAlias
orderBy[].sourceAlias

Restricao conhecida: pageSize <= 500.

12. Fluxo de bootstrap em nova maquina

  1. clonar o repositorio
  2. garantir OpenEdge e PASOE instalados
  3. ajustar paths locais
  4. revisar openedge.properties
  5. revisar temp/JNA
  6. confirmar os bancos disponiveis
  7. subir o PASOE
  8. abrir web/index.html
  9. configurar contexto em web/context-selector.html

14. Fluxo de desenvolvimento web

  1. tratar web/index.html como shell principal
  2. preservar o contexto no appbar
  3. evitar duplicar cliente/ambiente nas paginas internas
  4. manter selecao de empresa nas paginas de consulta
  5. usar context-manager.js para estado funcional
  6. usar menu-pages.json para navegacao

15. Fluxo de desenvolvimento ABL

  1. identificar se a alteracao afeta serializer, validacao, plano, execucao ou fila
  2. revisar impacto no contrato JSON
  3. lembrar que boConsDin.p e boMetaDados.p sao referencia
  4. se houver banco em -1, parar PASOE antes de usar _progres.exe
  5. reiniciar PASOE apenas quando a mudanca realmente exigir

16. Fluxo de troubleshooting rapido

Se a UI nao refletir o contexto esperado

  • revisar web/context-manager.js
  • revisar localStorage
  • revisar web/context-selector.html

Se o PASOE nao listar bancos esperados

  • revisar openedge.properties
  • confirmar stop/start completo
  • esperar o startup terminar antes de testar endpoint

Se _progres.exe falhar por base em single-user

  • garantir que o PASOE foi parado
  • nao rodar stop e abertura de banco em paralelo

Se o load de .d falhar

  • usar o dump-name sem extensao
  • apontar o diretorio correto
  • se surgir _File record not on file. (138), investigar mecanismo de load e nao apenas schema basico
